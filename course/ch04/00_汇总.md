# CH04 MCP 核心工作场景（下）- 优质作业汇总

> 本文档汇总了 CH04 笔记卡中质量最好的 4 篇作业，从不同角度解析了非文本场景下的 MCP 工具应用和 AI 时代的工作方式变化。

---

## 一、核心观点总结

### 1. AI 时代工作场景的新分类视角

在传统的"事务型 vs 沟通型"和"文本型 vs 非文本型"分类基础上，我们可以从**工作载体**和**工作性质**两个维度重新审视工作场景：

**工作载体维度：**
- **文本型**：围绕文字、数字、代码展开
- **非文本型**：涉及视觉、听觉、空间性内容

**工作性质维度：**
- **事务型**：有明确规则和流程，可标准化，强调效率
- **协作型**：需要多方参与、讨论、共创，强调沟通和创造力

这种分类不仅适用于工具选择，更重要的是揭示了**AI 对不同类型工作的影响差异**。

### 2. 非文本场景的工具生态现状

**非文本事务型场景**：工具链相对成熟，涵盖图表、图片、音频、视频、特殊文档等多个领域。

**非文本沟通型场景**：目前仍处于市场空白和萌芽阶段，但蕴含巨大未来潜力。

## 二、四象限工作场景详解

### 象限分布图

```
    ^ 协作型 (Collaborative)
    |
    |    象限②          |        象限④
    | 协作-文本型       |   协作-非文本型
    |  (PRD共写)       |  (头脑风暴)
    |
    |------------------+-------------------> 文本/非文本
    |
    |    象限①          |        象限③
    | 事务-文本型       |   事务-非文本型
    | (周报/脚本)       |  (PPT/剪辑)
    |
    v 事务型 (Transactional)
```

### 四象限特点对比

| 象限 | 类型 | 核心特点 | 典型场景 | AI 作用 |
|---|---|---|---|---|
| ① | 事务 - 文本 | 高重复、可独立完成 | 周报套版、合同校对 | **最易被自动化** |
| ② | 协作 - 文本 | 多方共创、迭代版本 | 共写 PRD、Code Review | **协作伙伴** |
| ③ | 事务 - 非文本 | 技能依赖、标准范式 | 套版 PPT、基础修图 | **生成式颠覆** |
| ④ | 协作 - 非文本 | 实时互动、激发创意 | 头脑风暴、战略规划 | **灵感催化剂** |

  - 积分有限

### 4. 视频类 MCP

#### 视频处理完整流程：素材获取 → 视频剪辑 → 视频分发

#### 4.1 视频素材获取

**油管下载器 MCP（yt-dlp-mcp）** ⭐⭐⭐⭐⭐
- **优点**：
  - 基于强大的开源项目 YT-DLP
  - 一键下载多平台视频
  - 支持 YouTube、B 站、小红书等
  - 可下载视频和字幕
- **缺点**：
  - 需预先安装 YT-DLP 工具（命令行）
  - 无法下载抖音视频
- **替代方案**：抖音视频可用"抖音小助手 MCP"
- **链接**：https://www.modelscope.cn/mcp/servers/@kevinwatt/yt-dlp-mcp

#### 4.2 视频剪辑

**视频剪辑 MCP（video-clip-mcp）** ⭐⭐⭐⭐
- **优点**：
  - 支持读取视频元数据
  - 根据明确时间点进行本地剪辑
  - 有效解决隐私和带宽问题
- **局限**：
  - 无法理解视频内容
- **进阶用法**：
  - 可结合多模态大模型（如 Gemini）
  - 让 AI 分析内容并确定剪辑节点
  - 实现完整自动化工作流
- **链接**：
  - GitHub: https://github.com/pickstar-2002/video-clip-mcp
  - 魔搭：https://www.modelscope.cn/mcp/servers/ModelScopePickStar/video-clip-mcp-MCP-Agent-Challenge

#### 4.3 视频分发

**现状**：暂时没有好用的视频分发自动化 MCP

### 5. 特殊文档类 MCP

#### 5.1 PPT 生成

**搞定 PPT MCP（okppt）** ⭐⭐⭐⭐
- **工作原理**：
  - AI 生成 SVG 矢量图形代码
  - 转换为 PPT 格式
- **优点**：
  - 既能生成图形，也能生成文字
- **缺点**：
  - 缺少常用固定布局
  - 依赖大模型的 SVG 代码创建能力
- **关键注意事项**：
  - ⚠️ 必须使用**微软官方 PowerPoint**打开
  - 使用"转化为形状"功能转为可编辑元素
  - **WPS Office 不支持此功能**
- **链接**：https://www.modelscope.cn/mcp/servers/@NeekChaw/okppt

#### 5.2 Excel 处理

**Excel MCP（excel-mcp-server）** ⭐⭐⭐⭐
- **功能**：
  - 复杂数据操作
  - 格式化处理（如填充颜色）
  - 创建动态数据透视表
- **局限**：
  - 成功率和效果高度依赖大模型能力
  - 复杂操作需要大模型对数据有深入理解
- **链接**：https://www.modelscope.cn/mcp/servers/codeyijun/excel-mcp-server

## 四、非文本沟通型场景：未来的机遇

### 场景特点

**定义**：以人为中心，使用非文本形式（语音、图像、视频）进行实时互动的场景。

**与事务型的区别**：重心不再是独立完成任务，而是以人为导向的实时交互。

### 典型场景分类

#### 1. 演示说明（单向信息流）
- 在线培训
- 汇报路演
- 产品演示

#### 2. 协作互动（多方实时交流）
- 头脑风暴
- 商务谈判
- 视频会议
- 客户拜访

### 技术现状

**当前状态**：市场空白，处于萌芽阶段

**技术空缺**：
- 缺乏实时干预和辅助系统
- 无法实时告知演讲者如何调整内容
- 缺少多方协作的 AI 辅助工具

### 未来潜力

**发展方向**：
1. AI 技术的进步
2. 机器人技术的融合
3. AR/VR技术的成熟

**生态需求**：需要完整的 MCP 生态系统来整合各种工具，实现复杂的实时互动功能。

**创业机会**：每一个未被 AI 触及的空白场景，都等待着开发者去探索和创造。

## 五、AI 对工作的影响与启示

### 1. 工作的流动性

**重要观察**：没有任何一个岗位只存在于一个象限。

**实际工作流程示例**：开发一个新功能

```
象限④（协作-非文本）
    ↓ 团队头脑风暴，白板草图
象限②（协作-文本）
    ↓ 共同撰写PRD文档
象限①（事务-文本）
    ↓ 工程师编写代码
象限③（事务-非文本）
    ↓ 设计师制作图标
象限④（协作-非文本）
    ↓ 产品演示和复盘
```

### 2. AI 对不同象限的冲击差异

#### 事务型工作（象限①和③）

**影响**：替代或极大增强

**价值转移**：
- 从"执行"转向"使用工具高效执行"
- 从"完成任务"转向"校验 AI 结果"

#### 协作型工作（象限②和④）

**影响**：赋能而非替代

**AI 角色**：
- 超级助理
- 创意伙伴

**人的核心价值**：
- 人与人之间的沟通
- 共情能力
- 判断力
- 建立共识

### 3. 未来知识工作者的价值

#### 核心竞争力转移

```
从：事务型能力（象限①、③）
↓
到：协作型能力（象限②、④）
```

#### 三大关键技能

1. **引导和组织高质量协作**
   - 特别是在象限④（协作 - 非文本）
   - 越来越有价值

2. **熟练驾驭 AI 工具**
   - 自动化事务型工作
   - 释放时间和精力

3. **专注协作和创造**
   - 将更多精力投入协作
   - 发挥创造力和判断力

## 六、工具选择与使用建议

### 1. 隐私安全考量

**需要上传数据的工具（慎用）：**
- mcp-server-chart（图表上传云端）
- 抖音小助手（语音上传）
- 趣丸千音（语音上传）

**本地处理的工具（推荐）：**
- 视频剪辑 MCP（本地剪辑）
- 其他本地化工具

**原则**：涉及保密数据或私密信息时，优先选择本地处理工具。

### 2. 工具链组合策略

#### 图片处理链
```
图表需求 → mcp-server-chart（公开数据）
图片生成 → ModelScope-Image-Generation-MCP
```

#### 视频处理链
```
素材获取 → 油管下载器MCP
内容理解 → Gemini等多模态大模型
视频剪辑 → video-clip-mcp
```

#### 文档处理链
```
PPT生成 → 搞定PPT MCP → PowerPoint转化
Excel处理 → Excel MCP（配合强大大模型）
```

### 3. 成本优化建议

**免费工具优先**：
- ModelScope-Image-Generation-MCP（免费但有次数限制）
- 油管下载器 MCP（完全免费）

**付费工具场景**：
- 趣丸千音：商用、批量语音生成时才考虑

### 4. 功能边界理解

**必须明确的问题**：
1. 工具能否理解内容？
2. 是否涉及云端处理？
3. 隐私风险有多大？
4. 配置有多复杂？
5. 成功率如何？

**示例**：
- 视频剪辑 MCP：不能理解内容 → 需配合多模态大模型
- 搞定 PPT MCP：依赖 SVG 代码能力 → 需要强大的大模型
- Excel MCP：复杂操作依赖大模型智商 → 选择高质量模型

## 七、实践工作流示例

### 示例 1：自动化视频剪辑工作流

```
1. 使用油管下载器MCP获取视频素材
   ↓
2. 使用Gemini等多模态大模型分析视频内容
   ↓
3. AI确定需要剪辑的时间节点
   ↓
4. 使用视频剪辑MCP执行本地剪辑
   ↓
5. 输出最终视频
```

**优势**：
- 全流程自动化
- 本地处理保护隐私
- 结合 AI 内容理解能力

### 示例 2：快速生成演示 PPT

```
1. 准备内容大纲和数据
   ↓
2. 使用大模型生成SVG代码
   ↓
3. 搞定PPT MCP转换为PPT
   ↓
4. 用PowerPoint打开并"转化为形状"
   ↓
5. 手动调整细节和布局
```

**注意事项**：
- 必须用微软 PowerPoint
- 生成的 PPT 可能需要手动调整
- 布局可能不如预期

### 示例 3：数据可视化流程

```
1. 准备数据（注意隐私）
   ↓
2a. 公开数据 → mcp-server-chart
2b. 保密数据 → MCP ECharts（本地）
   ↓
3. 生成图表
   ↓
4. 导出并集成到报告中
```

## 十、总结与展望

### 核心要点

1. **工作场景的多维度**：工作场景不是单一的，而是在四个象限中流动的。理解这种流动性，才能更好地应用 AI 工具。

2. **工具链的成熟度**：非文本事务型场景的工具链已相对成熟，可以满足大部分需求。但非文本沟通型场景仍是空白，充满机遇。

3. **隐私与效率的平衡**：在选择工具时，必须在隐私安全和使用效率之间找到平衡。涉及敏感数据时，优先选择本地处理工具。

4. **AI 的差异化影响**：AI 对事务型工作是替代，对协作型工作是赋能。理解这种差异，才能找到自己的价值定位。

5. **未来能力要求**：未来知识工作者的核心竞争力正在从事务型能力转向协作型能力。熟练驾驭 AI 工具、引导高质量协作，是关键技能。

### 行动建议

**立即可以做的：**
1. 根据自己的工作场景，选择合适的 MCP 工具
2. 建立自己的工具链和工作流
3. 注意隐私安全，选择合适的处理方式

**中期需要培养的：**
1. 熟练使用各类 MCP 工具
2. 学会组合工具解决复杂问题
3. 理解工具的功能边界

**长期需要转变的：**
1. 从事务执行者转变为工具驾驭者
2. 从独立工作者转变为协作组织者
3. 从技能专家转变为创意伙伴

### 未来展望

非文本沟通型场景虽然目前是空白，但随着 AI、机器人、AR/VR 技术的发展，这个领域必将迎来爆发。

每一个尚未被 AI 触及的空白场景，都等待着开发者去探索和创造。这既是挑战，也是机遇。
