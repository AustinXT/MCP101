## 【课程笔记】

### 1. MCP 生态的组成

MCP 生态由规则制定者、开发者、平台创建者以及用户构成。

- **规则制定者**：确立 MCP 协议
- **开发者**：在协议基础上开发 MCP 服务器
- **平台创建者**：提供 MCP 市场
- **用户**：使用 AI 软件

### 2. 普通用户接触最频繁的环节

作为普通用户，个人接触最频繁的是 MCP 市场和 AI 软件。

#### MCP 市场

**国内首推**：modelscope (https://www.modelscope.cn/mcp)

**国外推荐**：
- MCP.so (https://mcp.so)
- MCP Market (https://mcpmarket.com)
- smithery (https://smithery.ai) 等平台

**终极方案**（若市场没有合适 MCP）：
- Github 检索：关键词+MCP
- 或者 mcp+awesome

#### 调用 MCP 的 AI 软件

**AI 聊天工具**
- Claude desktop，cherry studio, Lingma
- 门槛低，但自定义能力有限

**AI 代码编辑器**
- Trae，Cursor，windsurf（课程首推）

**工作流平台**
- Dify，langchain，n8n
- 门槛较高，较高自定义能力

**纯代码实现**
- 自定义和困难程度最高

### 3. 筛选 MCP 工具的步骤

#### （1）拆解问题
将复杂任务拆解为具体的步骤/动作，然后根据具体的步骤作为关键词来搜索。拆解过程本身也可以借助大模型。

#### （2）多留备选
每个步骤/动作 保留 2-3 个工具，给后续比较筛选留出余量。

#### （3）评估筛选

- **功能匹配**：是否解决了核心问题
- **质量可信**：看使用量，点赞数，Github star 数，更新频率等
- **集成易用**：首选 uvx 或 mpx 命令一键安装的工具，对小白友好；而 uv 需要提前安装原始仓库，过程较复杂

## 【学习心得】

1. **个人定位与目标**：目前自己只能做一个普通用户，能安装清楚 MCP 工具以及熟练使用就算是通关第一步了，未来希望自己能尝试做一个开发者，做出自己第一个技术类的小作品。

2. **工具选择**：
   - MCP 市场：个人目前使用最多的是魔搭 modelscope，Github 一直很少去搜索，但作为一颗有学习技术的心，现在开始已经逐渐学习查阅 Github
   - AI 软件：目前个人使用最多的是 Cursor，其他的等自己技术逐渐变强且有具体需要时再去学习如何使用

3. **入门技巧**：作为入门，还有一个快速筛选 MCP 工具的方法就是"问大神"，直接问身边的技术高手，让对方推荐 ^_^
